Network layer security

Secure Sockets Layer (SSL) / Transport Layer Security (TLS) for web traffic

There are two distinct ways that a program can initiate a secure connection with a server:

  1. By Port (SSL): connection to a port that expects to the session to start with security negotiation e.g. port 443 for https (secure web), 993 for secure IMAP, 995 for secure POP, etc.
  2. By Protocol (TSL): connection where the program will connect “insecurely” first and use special commands to enable encryption e.g. “STARTTLS” used in outbound email (SMTP) connections.
